# KETRI 기존 사이트 데이터 추출 가이드

## 📋 개요

본 문서는 기존 KETRI 웹사이트에서 실제 데이터를 추출하여 새 사이트에 적용하는 방법을 안내합니다.

## 🔍 기존 사이트 상태 확인

### 1. 사이트 접근성 확인
현재 ketri.co.kr 도메인에 대한 접근을 시도했으나 연결할 수 없음:
- **HTTPS:** SSL 인증서 오류
- **HTTP:** 연결 실패
- **결론:** 현재 운영 중인 사이트가 없거나 접근 제한됨

### 2. 가능한 데이터 소스

#### A. 내부 자료 요청
다음 자료들을 회사 내부에서 확보 가능:
- **회사 소개 자료**
- **서비스 브로셔**
- **기존 웹사이트 백업**
- **연락처 정보 문서**
- **회사 로고 및 이미지**

#### B. 공개 정보 수집
- **사업자등록번호로 회사 정보 조회**
- **ISO 인증서 정보**
- **관련 기관 등록 정보**

## 🛠️ 데이터 추출 작업 계획

### Phase 1: 기본 정보 수집 (1-2일)
1. **회사 등기사항 확인**
   - 법인등기부등본으로 주소, 대표자 확인
   - 사업자등록증으로 업종, 등록번호 확인

2. **연락처 정보 수집**
   - 대표 전화번호, 이메일 확인
   - 부서별 담당자 연락처 수집

3. **인증 및 자격 정보**
   - ISO 인증서 스캔
   - 관련 기관 등록증 확인

### Phase 2: 콘텐츠 수집 (3-5일)
1. **회사 소개 자료**
   - 기존 브로셔, 소개서류 수집
   - 대표이사 인사말 원본 확보

2. **서비스 상세 정보**
   - 각 서비스별 상세 설명
   - 가격표 및 프로세스 정보
   - 실제 사례 연구 (Case Study)

3. **이미지 및 미디어**
   - 회사 로고 (AI, EPS, PNG 형식)
   - 사무실 사진
   - 장비 사진
   - 대표이사 사진

### Phase 3: 데이터 구조화 (2-3일)
1. **데이터베이스 설계**
   - Firestore 컬렉션 구조 정의
   - 데이터 타입 및 필드 정의

2. **콘텐츠 마이그레이션**
   - 기존 텍스트 콘텐츠 정리
   - 이미지 최적화 및 업로드
