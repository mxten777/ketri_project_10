# KETRI 웹사이트 리뉴얼 프로젝트 - 취지 및 설계서

## 📋 프로젝트 개요

### 프로젝트명
**한국환경안전연구소 (KETRI) 웹사이트 리뉴얼 프로젝트**

### 기간
2025년 1월 ~ 2025년 11월 (11개월)

### 목표
글로벌 Top-tier 연구기관 수준의 프리미엄 웹사이트 구축을 통해 브랜드 가치 상승 및 고객 신뢰도 향상

## 🎯 프로젝트 취지

### 1. 비전 및 목표

**"첨단 환경·안전 솔루션의 선두주자로서 미래를 보호하는 글로벌 리더"**

- **브랜드 리포지셔닝:** 국내 최고 수준 → 글로벌 Top-tier 연구기관으로 도약
- **디지털 트랜스포메이션:** 전통적인 연구소 이미지 → 혁신적인 테크 기업으로 변화
- **고객 경험 혁신:** 단순 정보 제공 → Interactive한 서비스 플랫폼으로 발전

### 2. 시장 환경 분석

#### 산업 트렌드
- **디지털 전환 가속화:** 환경안전 분야에서도 DX 필수
- **고객 기대치 상승:** Apple/Google 수준의 UX 요구
- **경쟁 심화:** 스타트업 및 글로벌 기업의 시장 진입

#### 경쟁사 벤치마킹
- **Apple:** 심플함, 직관성, 프리미엄 경험
- **Stripe:** 개발자 친화적, API 중심 설계
- **MIT:** 혁신성, 교육적 접근, 커뮤니티 구축

#### KETRI 현황
- **강점:** 30년 전문성, ISO 인증, 풍부한 실적
- **약점:** 오래된 웹사이트, 낮은 디지털 접근성
- **기회:** 환경안전 시장 성장, 정부 정책 지원
- **위협:** 신규 경쟁자 등장, 고객 이탈

### 3. 문제점 진단

#### 현재 웹사이트 이슈
- **UI/UX:** 90년대 수준의 디자인, 모바일 미지원
- **기능:** 정적 정보 제공만 가능, 상호작용 부족
- **접근성:** 웹 표준 미준수, 스크린 리더 미지원
- **성능:** 느린 로딩 속도, SEO 최적화 부족

#### 고객 피드백
- "현대적인 느낌이 부족하다"
- "모바일에서 사용하기 불편하다"
- "원하는 정보를 찾기 어렵다"
- "신뢰할 수 있는 이미지가 아니다"

## 🎨 디자인 설계 원칙

### 1. 핵심 가치

#### 프리미엄 (Premium)
- **색상 팔레트:** 전문적이고 신뢰감을 주는 블루 계열
- **타이포그래피:** Inter + Playfair Display (모던 + 전통)
- **시각 요소:** Glass morphism, Gradient, Shadow 효과

#### 신뢰성 (Trust)
- **데이터 시각화:** 정확한 정보 전달
- **인증 마크:** ISO, 공인 기관 배지 강조
- **고객 후기:** 실제 사례 기반 스토리텔링

#### 혁신성 (Innovation)
- **기술 도입:** 최신 웹 기술 (React 18, TypeScript)
- **인터랙션:** 마이크로 애니메이션, 스크롤 트리거
- **사용자 경험:** 직관적 네비게이션, 스마트 검색

### 2. 디자인 시스템

#### 색상 시스템
```css
/* Primary Colors */

/* Neutral Colors */
--gray-50: #f9fafb;      /* 가장 연한 회색 */
```

#### 컴포넌트 라이브러리
- **Button:** Primary, Secondary, Ghost variants
- **Card:** Glass effect, Shadow variants
- **Form:** Validation states, Accessibility labels
- **Navigation:** Desktop dropdown, Mobile sidebar

### 3. 사용자 경험 (UX) 설계

#### 사용자 페르소나
1. **기업 고객 (B2B)**
   - 산업안전 담당자, 환경관리 책임자
   - 정보 탐색 → 견적 요청 → 계약 체결

2. **개인 고객 (B2C)**
   - 건물주, 입주민, 학생
   - 서비스 안내 → 예약 → 결과 확인

3. **기관 고객 (B2G)**
   - 정부 기관, 공공 단체
   - 입찰 정보 → 제안서 제출 → 계약

#### 사용자 여정 (User Journey)
```
인식 → 관심 → 고려 → 구매 → 충성도 → 추천
   ↓     ↓     ↓     ↓     ↓     ↓
랜딩 → 탐색 → 비교 → 신청 → 이용 → 공유
```

#### 주요 UX 패턴
- **Progressive Disclosure:** 정보 단계적 노출
- **Visual Hierarchy:** 중요도에 따른 시각적 강조
- **Feedback System:** 로딩 상태, 성공/실패 메시지
- **Error Prevention:** 폼 유효성 검증, 확인 단계

## 🛠️ 기술 아키텍처

### 1. 기술 스택 선정 이유

#### Frontend Framework
**React 18 + TypeScript**
- **이유:** 컴포넌트 재사용성, 타입 안정성, 생태계 풍부
- **대안 고려:** Vue.js, Svelte, SolidJS
- **선정 근거:** 팀 숙련도, 라이브러리 지원, 미래 확장성

#### Build Tool
**Vite**
- **이유:** 초고속 개발 서버, 최적화된 프로덕션 빌드
- **장점:** HMR, 트리 쉐이킹, 플러그인 생태계

#### Styling
**Tailwind CSS**
- **이유:** 유틸리티 퍼스트, 일관된 디자인 시스템
- **장점:** 커스터마이징 용이, 파일 크기 최적화

#### Animation
**Framer Motion**
- **이유:** 선언적 애니메이션, React 생태계 통합
- **장점:** 성능 최적화, 복잡한 애니메이션 지원

#### Backend
**Firebase**
- **이유:** 서버리스, 실시간 데이터베이스, 인증
- **장점:** 개발 속도, 확장성, 비용 효율성
